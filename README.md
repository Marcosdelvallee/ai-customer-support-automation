##  What It Does

A customer submits a support request through the web portal. Within **2 seconds**:

- The message is **classified** by Groq's Llama 3.3 70b across 8 dimensions
- **Simple tickets** get an AI-generated response sent directly to their email
- **Critical or complex tickets** are escalated to the support team via Slack with full context
- Every ticket is **persisted** in PostgreSQL with a complete audit trail
- The **agent dashboard** shows real-time analytics from Supabase

No human intervention required for ~74% of incoming tickets.

##  Key Features

**AI Classification** — Every ticket is classified with 8 fields: category (billing, technical, account, complaint, general, sales, refund, shipping), urgency (low/medium/high/critical), sentiment (positive/neutral/negative/very_negative), complexity (simple/moderate/complex), language (ISO 639-1 auto-detection), one-sentence summary, needs_human flag, and confidence score.

**Auto-Response** — Simple and moderate tickets receive a professional, empathetic response generated by Llama 3.3 70b in the customer's detected language. Delivered via email through Resend.

**Smart Escalation** — Complex tickets, critical urgency, or those requiring human attention trigger an immediate Slack alert with the full ticket context and are saved to Supabase with status `escalated`.

**Multi-language** — Language is auto-detected by the LLM. Responses are generated in the same language as the customer's message (tested with EN, ES, PT).

**Real-time Dashboard** — Agent-facing dashboard pulls live data from Supabase. Shows KPIs, category/sentiment/urgency charts, and a ticket table. Clicking any row opens a full detail modal with classification, confidence score, and the response sent to the customer.

**Customer Portal** — Clean, bilingual (EN/ES) form for submitting support requests. Confirms submission with Ticket ID.

##  Setup

### Prerequisites
- Node.js v18+ — verify with `node --version`
- Free accounts: [Groq](https://console.groq.com) · [Supabase](https://supabase.com) · [Slack](https://slack.com) · [Resend](https://resend.com)

### 1. Database — Supabase
```
1. Create project at supabase.com (free, no credit card)
2. SQL Editor → New query → paste schema.sql → Run
3. Save: Settings → Database → Host + Password
4. Save: Settings → API → anon public key
```

### 2. Groq API Key
```
1. console.groq.com → API Keys → Create API Key (free, no credit card)
2. Save the key (starts with gsk_...)
```

### 3. Slack App
```
1. api.slack.com/apps → Create New App → From scratch
2. OAuth & Permissions → Bot Token Scopes → add: chat:write, chat:write.public
3. Install to Workspace → copy Bot User OAuth Token (xoxb-...)
4. Create channel #support-alerts → copy Channel ID
5. In the channel: /invite @YourBotName
```

### 4. Resend
```
1. resend.com → API Keys → Create API Key (free, no credit card)
2. Save the key (starts with re_...)
3. Verify your email address under Emails for sandbox testing
```

### 5. Start n8n
```bash
# Mac / Linux
NODE_FUNCTION_ALLOW_BUILTIN=https,http npx n8n

# Windows PowerShell
$env:NODE_FUNCTION_ALLOW_BUILTIN="https,http"; npx n8n
```

Open `http://localhost:5678` — first run downloads ~400 MB, subsequent runs are instant.

### 6. Import & Configure Workflow
```
1. n8n → Workflows → Import from File
   → select ai-customer-support-workflow-GROQ.json

2. Credentials to configure:
   ┌─────────────────────────────────────────────────────────────┐
   │ Node                    │ Credential type  │ Value          │
   ├─────────────────────────┼──────────────────┼────────────────┤
   │ Groq: Classify Ticket   │ Header Auth      │ Authorization  │
   │ Groq: Generate Response │                  │ Bearer gsk_... │
   ├─────────────────────────┼──────────────────┼────────────────┤
   │ Supabase nodes (×2)     │ PostgreSQL       │ host/pass      │
   ├─────────────────────────┼──────────────────┼────────────────┤
   │ Slack: Alert            │ Slack API        │ xoxb-...       │
   └─────────────────────────┴──────────────────┴────────────────┘

3. In Slack node → replace C_SUPPORT_ALERTS with your Channel ID
4. In Code nodes → replace SUPABASE_REF and SUPABASE_KEY
5. In Resend node → replace RESEND_KEY with your re_... key
6. Save → toggle Active
```

##  Known Limitations & Roadmap

**Current limitations:**
- Inbound email replies are not yet handled. When a customer replies to the AI response, it reaches Resend's sandbox. Production requires a custom domain with inbound webhook configuration.
- n8n runs locally via `npx`. For 24/7 availability, deploy to Railway or n8n Cloud.

**Planned enhancements:**
- [ ] Inbound email reply handling with custom domain
- [ ] Vector embeddings + knowledge base for context-aware responses
- [ ] SLA tracking and breach alerts
- [ ] Agent assignment and ticket ownership workflow
- [ ] Stripe webhook integration for automatic billing ticket creation
- [ ] Fine-tuning classifier on historical ticket data

---

##  Design Decisions

**Why n8n over custom code?** n8n provides a visual workflow immediately understandable to non-technical stakeholders while allowing full customization via Code nodes. The automation logic is auditable, maintainable, and easy to extend.

**Why Groq + Llama over OpenAI?** Zero cost with equivalent quality. Demonstrates awareness of the broader LLM ecosystem beyond OpenAI and a cost-optimization mindset — essential for production automation engineering.

**Why two separate frontends?** The customer portal and agent dashboard serve different audiences with different trust levels. Combining them would expose internal metrics to customers and create a confusing UX.

##  Funcionalidades Principales

**Clasificación con IA** — Cada ticket se clasifica con 8 campos: categoría (billing, technical, account, complaint, general, sales, refund, shipping), urgencia (low/medium/high/critical), sentimiento (positive/neutral/negative/very_negative), complejidad (simple/moderate/complex), idioma (detección automática ISO 639-1), resumen en una oración, flag needs_human y score de confianza.

**Respuesta Automática** — Los tickets simples y moderados reciben una respuesta profesional y empática generada por Llama 3.3 70b en el idioma detectado del cliente. Se entrega por email a través de Resend.

**Escalamiento Inteligente** — Los tickets complejos, de urgencia crítica o que requieren atención humana disparan una alerta inmediata en Slack con el contexto completo del ticket y se guardan en Supabase con estado `escalated`.

**Multi-idioma** — El idioma se detecta automáticamente por el LLM. Las respuestas se generan en el mismo idioma del mensaje del cliente (probado con EN, ES, PT).

**Dashboard en Tiempo Real** — Dashboard para el equipo interno que obtiene datos en vivo de Supabase. Muestra KPIs, gráficos de categorías/sentimiento/urgencia y tabla de tickets. Al hacer click en cualquier fila se abre un modal con la clasificación, score de confianza y la respuesta enviada al cliente.

**Portal del Cliente** — Formulario limpio y bilingüe (EN/ES) para enviar consultas. Confirma el envío con el Ticket ID.

##  Decisiones de Diseño

**¿Por qué n8n en lugar de código custom?** n8n provee un workflow visual inmediatamente comprensible para stakeholders no técnicos, mientras permite personalización completa via nodos Code. La lógica de automatización es auditable, mantenible y fácil de extender.

**¿Por qué Groq + Llama en lugar de OpenAI?** Costo cero con calidad equivalente. Demuestra conocimiento del ecosistema de LLMs más allá de OpenAI y una mentalidad de optimización de costos — una habilidad clave para automation engineering en producción.

**¿Por qué dos frontends separados?** El portal del cliente y el dashboard de agentes atienden a audiencias distintas con distintos niveles de confianza y necesidades de información. Combinarlos expondría métricas internas a los clientes y crearía una UX confusa.
